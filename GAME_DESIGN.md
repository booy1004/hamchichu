# 🐹 왁왁왁! 치즈 먹방 애햄이 - 게임 기획서

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 퍼즐 (지뢰찾기 변형) |
| 플랫폼 | 웹 (PC + 모바일) |
| 기술 스택 | React + Vite + framer-motion |
| 타겟 유저 | 캐주얼 게임 유저, 귀여운 거 좋아하는 사람 |
| 컨셉 | 햄스터 리쿠가 치즈 창고에서 고양이를 피해 치즈를 먹는 지뢰찾기 |

### 스토리
> 치즈 창고에 잠입한 햄스터 리쿠!
> 하지만 창고 곳곳에 고양이들이 숨어서 자고 있다...
> 고양이를 깨우지 않고 모든 치즈를 먹어치워라! 🧀

---

## 2. 세계관 & 캐릭터

### 리쿠 (플레이어 캐릭터)
- 종족: 골든 햄스터
- 성격: 먹보, 용감함, 볼따구에 치즈 저장하는 습관
- 특기: 왁왁왁 치즈 먹방
- 상태 변화:
  - 대기: 두리번두리번 (idle 애니메이션)
  - 이동: 뚜벅뚜벅 기어감
  - 먹기: 왁왁왁! (볼따구 점점 빵빵)
  - 놀람: 끼이익!! (고양이 발견 시)
  - 클리어: 엄지척 + 배 통통

### 고양이 (지뢰)
- 종류: 주황이, 회색이, 까만이, 금색이, 크림이 (5종)
- 평소: 치즈 속에 숨어서 자는 중 (zzz...)
- 발견 시: "애옹!" 하고 벌떡 일어남

### 해바라기씨 (깃발)
- 리쿠가 위험 지역에 표시하는 마커
- 고양이가 있을 것 같은 칸에 꽂아둠

## 3. 게임 규칙

### 기본 규칙 (지뢰찾기와 동일)
1. 격자형 보드에 고양이(지뢰)가 랜덤 배치
2. 칸을 클릭하면 리쿠가 해당 칸의 치즈를 먹음
3. 먹은 칸에는 주변 고양이 수가 숫자로 표시
4. 숫자가 0이면 주변 칸이 연쇄로 열림
5. 고양이 칸을 클릭하면 게임 오버
6. 모든 안전한 칸을 열면 클리어

### 조작법

| 플랫폼 | 조작 | 동작 |
|--------|------|------|
| PC | 좌클릭 | 치즈 먹기 (칸 열기) |
| PC | 우클릭 | 해바라기씨 놓기/빼기 (깃발) |
| PC | 더블클릭 | 코드 - 주변 자동 오픈 |
| 모바일 | 탭 | 치즈 먹기 |
| 모바일 | 꾹 누르기 (0.4초) | 해바라기씨 놓기/빼기 |
| 모바일 | 더블탭 | 코드 - 주변 자동 오픈 |

### 코드(Chord) 기능
- 이미 열린 숫자 칸을 더블클릭
- 주변 해바라기씨 수 = 숫자일 때 나머지 칸 자동 오픈
- 해바라기씨를 잘못 놓았으면 게임 오버

### 난이도

| 난이도 | 보드 크기 | 고양이 수 | 아이콘 |
|--------|----------|----------|--------|
| 쉬움 | 9 × 9 | 10마리 | 🐣 |
| 보통 | 16 × 16 | 40마리 | 🐥 |
| 어려움 | 16 × 30 | 99마리 | 🐔 |

## 5. 애니메이션 시나리오

### 5-1. 안전한 칸 클릭
```
1. 리쿠가 클릭한 칸으로 이동 (뚜벅뚜벅)
2. 칸 도착 → 왁왁왁 먹기 모션
3. 치즈 타일이 뒤집히며 숫자 표시
4. 리쿠 볼따구 약간 커짐
5. 말풍선: "왁왁왁!"
```

### 5-2. 빈 칸 연쇄 오픈 (숫자 0)
```
1. 리쿠 왁왁왁 먹기
2. 주변 칸들이 도미노처럼 연쇄로 뒤집힘
3. 뒤집히는 애니메이션에 약간의 딜레이 (물결 효과)
```

### 5-3. 해바라기씨 설치
```
1. 해바라기씨가 위에서 톡! 떨어지며 꽂힘
2. 스프링 바운스 애니메이션
```

### 5-4. 고양이 발견 (게임 오버)
```
1. 클릭한 칸에서 고양이가 "애옹!" 팝업
2. 리쿠 눈 커지며 "끼이익!!" 말풍선
3. 리쿠가 먹던 치즈 뿜기 (파티클)
4. 리쿠 뒤로 데굴데굴 굴러감
5. 모든 고양이 위치 공개
6. 화면 어두워지며 게임오버 오버레이
7. 🙀 "고양이한테 들켰다!" 메시지
```

### 5-5. 게임 클리어
```
1. 마지막 칸 오픈
2. 리쿠 볼따구 최대 빵빵 상태
3. 컨페티 (색종이) 터짐
4. 리쿠 엄지척 → 하트 뿅
5. 배 통통 두드리기
6. "치즈 먹방 성공!" 오버레이
7. 클리어 시간 표시
8. 이름 입력 → 기록 저장
```

## 7. 대결 모드

### 개요
같은 지뢰 배치의 맵으로 친구와 클리어 타임을 비교하는 모드

### 흐름
```
1. 플레이어 A: "대결 코드 만들기" 클릭
2. 6자리 숫자 코드 생성 (예: 142857)
   - 첫 자리: 난이도 (1=쉬움, 2=보통, 3=어려움)
   - 나머지 5자리: 맵 시드
3. 코드를 친구에게 전달 (복사 버튼)
4. 플레이어 B: "코드로 참가하기" → 코드 입력
5. 동일한 맵으로 플레이
6. 각자 클리어 타임 비교
```

### 기술 구현
- 시드 기반 의사 난수 생성기 (mulberry32)
- 같은 시드 → 같은 지뢰 배치 보장
- 서버 불필요 (코드만 공유하면 됨)

## 10. 에셋 제작 가이드

### 추천 툴
- **힉스필드 (Hitsfield)**: 캐릭터 애니메이션 제작
- **Aseprite**: 픽셀아트 스프라이트
- **Spine / Live2D**: 실시간 캐릭터 모션 (고급)
- **After Effects → Lottie**: 벡터 애니메이션

### 렌더링 규격
- 투명 배경 필수 (PNG / WebM / APNG)
- 셀 크기 기준: 36px (쉬움) ~ 28px (모바일)
- 에셋 원본은 128×128 이상으로 제작 후 축소 권장
- WebM: VP9 코덱 + 알파 채널 (투명 배경 영상)

### 적용 방법
1. `public/assets/` 폴더에 에셋 파일 배치
2. `src/assets.js`에서 경로 연결
3. 컴포넌트가 자동으로 에셋 사용 (CSS 폴백 → 에셋 교체)

```javascript
// src/assets.js 예시
const ASSETS = {
  hamster: {
    idle: '/assets/hamster-idle.webm',
    eating: '/assets/hamster-eat.webm',
    shocked: '/assets/hamster-shocked.webm',
    clear: '/assets/hamster-clear.webm',
  },
  // ...
};
```
