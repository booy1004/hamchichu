# 🐹 뿌이뿌이! 파프리카 먹방 사쿠야 - 게임 기획서

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 퍼즐 (사과게임 변형) |
| 플랫폼 | 웹 (PC + 모바일) |
| 기술 스택 | React + Vite + framer-motion (기존 프로젝트 공유) |
| 타겟 유저 | 캐주얼 게임 유저, 귀여운 거 좋아하는 사람 |
| 컨셉 | 기니피그 사쿠야가 과수원에서 숫자 합 10을 만들어 먹이를 먹는 퍼즐 |
| 위치 | 햄스터 치즈먹방과 같은 사이트, 탭으로 전환 |

### 스토리
> 과수원에 놀러온 기니피그 사쿠야!
> 여기저기 맛있는 것들이 가득한데...
> 합이 10이 되는 것들을 골라서 먹어치워라! 뿌이뿌이~ 🫑

## 2. 세계관 & 캐릭터

### 사쿠야 (플레이어 캐릭터)
- 종족: 기니피그 (아비시니안 or 크레스티드)
- 성격: 먹보, 겁쟁이인데 먹이 앞에선 용감, 뿌이뿌이 수다쟁이
- 특기: 뿌이뿌이 파프리카 먹방, 팝콘 점프
- 상태 변화:
  - 대기: 코 벌름벌름 (idle 애니메이션)
  - 먹기: 아삭아삭! (입 오물오물)
  - 신남: 팝콘 점프! (통통 튀어오름)
  - 실패: 뒤돌아 앉기 (삐짐)
  - 클리어: 배 깔고 드러눕기 (만족)

## 3. 게임 규칙

### 기본 규칙 (사과게임 변형)
1. 격자형 보드에 1~9 숫자가 랜덤 배치
2. 인접한 셀들을 드래그로 선택 (상하좌우 + 대각선)
3. 선택한 숫자의 합이 정확히 10이면 해당 셀들이 사라짐
4. 셀이 사라지면 위의 셀들이 중력으로 떨어짐
5. 빈 열은 옆으로 당겨짐 (선택사항)
6. 제한 시간 내 최대한 높은 점수 획득
7. 더 이상 10을 만들 수 없거나 시간 종료 시 게임 끝

### 조작법

| 플랫폼 | 조작 | 동작 |
|--------|------|------|
| PC | 드래그 | 인접한 셀 선택 (경로 따라) |
| PC | 클릭 | 단일 셀 선택/해제 |
| 모바일 | 터치 드래그 | 인접한 셀 선택 |
| 모바일 | 탭 | 단일 셀 선택/해제 |

### 인접 규칙
- 선택하는 셀들은 서로 인접해야 함 (상하좌우 + 대각선 8방향)
- 드래그 경로를 따라 연결된 셀만 선택 가능
- 이미 선택한 셀을 다시 지나가면 선택 해제 (되돌리기)

### 난이도

| 난이도 | 보드 크기 | 제한 시간 | 파프리카 출현율 | 아이콘 |
|--------|----------|----------|---------------|--------|
| 쉬움 | 5 × 8 | 120초 | 높음 | 🌱 |
| 보통 | 6 × 10 | 90초 | 보통 | 🌿 |
| 어려움 | 8 × 12 | 60초 | 낮음 | 🌳 |

## 4. 점수 시스템

### 파프리카 보너스

각 셀에는 랜덤으로 파프리카가 배치될 수 있음. 파프리카가 있는 셀은 점수가 높음!

| 셀 종류 | 셀 배경색 | 점수 | 출현 확률 |
|---------|----------|------|----------|
| 기본 (파프리카 없음) | 연한 크림색 | 1점 | ~60% |
| 🟢 초록 파프리카 | 연두 틴트 | 2점 | ~15% |
| 🟡 노랑 파프리카 | 노랑 틴트 | 3점 | ~12% |
| 🟠 주황 파프리카 | 주황 틴트 | 4점 | ~8% |
| 🔴 빨강 파프리카 | 빨강 틴트 | 5점 | ~5% |

- 셀에는 숫자만 크게 표시, 점수는 셀에 안 씀
- 파프리카가 있는 셀은 배경에 파프리카 그림이 살짝 보임
- 화면 하단에 범례로 점수 안내 표시

### 점수 계산
합이 10이 되면 → 선택한 각 셀의 점수를 합산

예시:
- 3(기본) + 7(빨강🔴) = 10 → 1 + 5 = **6점**
- 2(노랑🟡) + 3(주황🟠) + 5(기본) = 10 → 3 + 4 + 1 = **8점**
- 1(빨강🔴) + 9(빨강🔴) = 10 → 5 + 5 = **10점** (잭팟!)

### 콤보 보너스
- 2콤보: 점수 × 1.5
- 3콤보: 점수 × 2
- 4콤보 이상: 점수 × 3
- 콤보는 3초 이내 연속 매치 시 유지

### 범례 UI (화면 하단)

```
┌──────────────────────────────────────────┐
│  📋 점수표                                │
│  ⬜ 기본 1점  🟢 2점  🟡 3점  🟠 4점  🔴 5점  │
└──────────────────────────────────────────┘
```

## 6. 애니메이션 시나리오

### 6-1. 셀 선택 중 (드래그)
```
1. 셀 위로 드래그하면 셀이 살짝 커지며 하이라이트
2. 선택된 셀들 사이에 연결선 표시
3. 현재 합계가 셀 근처에 떠다님 (예: "합: 7")
4. 합이 10이 되면 → 초록색 반짝 + "뿌이!" 표시
5. 합이 10을 넘으면 → 빨간색 흔들림
```

### 6-2. 매치 성공 (합 = 10)
```
1. 선택된 셀들이 "퐁!" 하고 터짐
2. 파프리카가 있던 셀은 파프리카가 사쿠야 입으로 날아감
3. 사쿠야 아삭아삭 먹기 모션
4. 점수 팝업 (+6점 같은)
5. 사쿠야 말풍선: "뿌이뿌이~"
6. 위의 셀들이 중력으로 떨어짐 (통통 바운스)
```

### 6-3. 콤보 발생
```
1. 콤보 카운터 숫자 커지며 반짝
2. 사쿠야 점점 신남 → 팝콘 점프 모션!
3. 5콤보 이상: 사쿠야 눈 반짝반짝 + "뿌이뿌이뿌이뿌이!!!"
4. 화면 테두리 살짝 반짝이는 이펙트
```

### 6-4. 시간 종료 / 매치 불가
```
1. 타이머 빨간색으로 깜빡
2. "시간 종료!" or "더 이상 먹을 수 없어!"
3. 사쿠야 뒤돌아 앉기 (삐짐 포즈)
4. 말풍선: "뿌이..." (시무룩)
5. 최종 점수 오버레이
```

### 6-5. 하이스코어 달성
```
1. 사쿠야 배 깔고 드러눕기
2. 만족스러운 표정 + 배 통통
3. 컨페티 터짐
4. "뿌이뿌이~ 배불러!" 말풍선
5. 이름 입력 → 기록 저장
```

## 7. 사쿠야 배불뚝이 시스템

1탄의 볼따구 시스템처럼, 사쿠야는 점수가 올라갈수록 배가 불러짐!

| 점수 구간 | 배 상태 | 표시 |
|-----------|---------|------|
| 0~25% | 날씬 | 기본 체형 |
| 25~50% | 약간 통통 | 배 살짝 커짐 |
| 50~75% | 통통 | 뒹굴뒹굴 체형 |
| 75~100% | 최대 먹보 | 배 빵빵 + 눈 반감김 |

- CSS scale 애니메이션으로 배 크기 변화
- 스프링 물리 적용 (통통 튀는 느낌)

## 8. 대결 모드

### 개요
같은 보드 배치로 친구와 점수를 비교하는 모드

### 흐름
```
1. 플레이어 A: "대결 코드 만들기" 클릭
2. 6자리 숫자 코드 생성
   - 첫 자리: 난이도 (1=쉬움, 2=보통, 3=어려움)
   - 나머지 5자리: 맵 시드
3. 코드를 친구에게 전달 (복사 버튼)
4. 플레이어 B: "코드로 참가하기" → 코드 입력
5. 동일한 보드로 플레이
6. 각자 최종 점수 비교
```

### 기술 구현
- 1탄과 동일한 시드 기반 의사 난수 생성기 (mulberry32)
- 같은 시드 → 같은 숫자 배치 + 같은 파프리카 배치 보장
- 서버 불필요 (코드만 공유)

## 11. 핵심 로직 (사과게임)

### 보드 생성
```
1. rows × cols 격자 생성
2. 각 셀에 1~9 랜덤 숫자 배정
3. 각 셀에 파프리카 랜덤 배정 (확률 기반)
4. 시드 기반이면 seededRandom 사용
```

### 드래그 선택
```
1. 마우스/터치 다운 → 시작 셀 선택
2. 드래그하며 인접 셀 추가 (8방향)
3. 이미 선택된 셀 위로 돌아가면 해제
4. 현재 합계 실시간 표시
5. 마우스/터치 업 → 합이 10이면 매치, 아니면 취소
```

### 중력 시스템
```
1. 매치된 셀 제거
2. 각 열에서 빈 칸 위의 셀들을 아래로 이동
3. 낙하 애니메이션 (바운스)
4. 완전히 빈 열은 유지 (옆으로 안 당김 - 심플하게)
```

### 게임 종료 판정
```
1. 타이머 0 → 종료
2. 매 턴마다 가능한 조합 존재 여부 체크
3. 가능한 조합 없으면 → "더 이상 먹을 수 없어!" → 종료
```
